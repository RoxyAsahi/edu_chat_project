# Plan: Fix Topic Action Menu UI and Restore Bento Grid Design

## Analysis of the Issue
1. **Left Sidebar Bento Style**: The user requested a Bento Grid style for the left sidebar with two distinct white cards for "Agents" and "Topics". Previous overrides had made these transparent and removed padding, causing elements to be cut off at the edges.
2. **Crowded Topic Menu**: The floating action menu for topics (appearing on hover) is "ugly and crowded". Analysis shows the menu uses text labels ("Rename", "Mark Unread", etc.) inside 30px circular buttons, causing text to overlap and overflow.

## Steps to Fix

### Part 1: Restore Bento Grid Layout
1. **Clean up `style.css`**: Remove any remaining `/* De-containerize the Navigation Suite */` style logic or associated list-item overrides that conflict with the white bento panel design.
2. **Restore Panel Padding**: Ensure panels have their default `20px` padding so headers and buttons aren't cut off.

### Part 2: Fix Crowded Topic Menu
1. **Update `renderer.js`**:
   - Replace text labels in `.topic-action-btn` with Material Symbols icons.
   - Use `edit` for Rename.
   - Use `mark_chat_unread` or `drafts` for Toggle Unread.
   - Use `lock` and `lock_open` for Toggle Lock.
   - Use `delete` for Delete.
   - Add `title` attributes (Rename, Mark Unread, etc.) to each button for accessibility/tooltips.
2. **Refine CSS in `style.css`**:
   - Clean up conflicting `.topic-actions` definitions.
   - Ensure the menu has a clean, horizontal layout with sufficient `gap` (e.g., `4px`).
   - Adjust the translucent background and blur to ensure readability against the list item.

### Part 3: Verification
1. Check both light and dark themes.
2. Verify that the topic title correctly truncates when the menu is visible.

## Summary
By switching to icons for the topic menu and restoring the containerized bento panels, the UI will regain its professional look, fix the "cut off" bug, and resolve the crowded menu issue.
