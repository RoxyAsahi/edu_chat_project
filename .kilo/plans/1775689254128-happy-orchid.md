# 打包 VCPChatLite 为 exe 文件的实施计划

## 概述
当前项目是一个标准的 Electron 应用。要将其打包成 Windows 可执行文件（`.exe`），最主流、功能最完善的工具是 **`electron-builder`**。它支持将应用打包为标准的 Windows 安装程序（NSIS）或单文件绿色版（Portable）。

在正式打包前，代码层面有一个**必须解决的关键问题：数据读写路径**。
当前 `main.js` 中使用 `path.join(__dirname, 'AppData')` 来存储数据。在应用被打包后，所有的代码和文件会被压缩进一个名为 `app.asar` 的只读包中（或者安装在 `C:\Program Files` 这样需要管理员权限的目录下）。如果打包后继续试图向 `__dirname` 写入数据，会导致应用报错或崩溃。

## 具体的实施步骤

### 1. 修改数据存储路径适配生产环境
我们需要修改 `main.js` 中的路径初始化逻辑。利用 `app.isPackaged` 来区分开发环境和打包后的生产环境：
- **开发环境**：保持现有逻辑，使用项目内的 `AppData` 目录。
- **生产环境（打包后）**：使用 Electron 推荐的系统级用户数据目录（通过 `app.getPath('userData')` 获取，通常位于 `%APPDATA%\vcpchat-lite`）。

### 2. 安装打包依赖
使用 npm 安装 `electron-builder` 作为开发依赖：
```bash
npm install --save-dev electron-builder
```

### 3. 配置 package.json
在项目的 `package.json` 中添加构建配置（`build` 字段）和对应的运行脚本（`scripts`）：

```json
{
  "scripts": {
    "start": "node scripts/launch-electron.js .",
    "build:win": "electron-builder --win",     // 打包 Windows 安装包
    "build:dir": "electron-builder --dir"      // 仅生成免安装目录，用于快速测试打包效果
  },
  "build": {
    "appId": "com.vcpchat.lite",
    "productName": "VCPChat Lite",
    "directories": {
      "output": "release"
    },
    "win": {
      "target": [
        "nsis",       // 生成常规安装程序
        "portable"    // 生成单文件绿色便携版
      ]
      // "icon": "build/icon.ico" // 如果有图标，可以取消这行注释并准备图标文件
    },
    "nsis": {
      "oneClick": false,                             // 允许用户自己点击“下一步”
      "allowToChangeInstallationDirectory": true     // 允许用户自定义安装路径
    }
  }
}
```

### 4. （可选）准备应用图标
如果你希望应用有自己的图标，需要制作一张 `256x256` 像素以上的 `.ico` 格式图片，放入项目中（比如放在新建的 `build` 目录中），并在 `package.json` 中配置引用。

### 5. 执行打包命令
配置完成后，在命令行运行：
```bash
npm run build:win
```
命令执行完毕后，在项目生成的 `release` 文件夹下就能找到编译好的 `.exe` 文件了。
