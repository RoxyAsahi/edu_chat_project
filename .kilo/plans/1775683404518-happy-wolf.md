# UI 重建计划：Bento Grid + Material Design (NotebookLM 风格)

## 目标
将 VCPChatLite 的用户界面重构为类似 Google NotebookLM 的风格。核心采用三栏式布局，结合 Bento Grid（便当盒网格）和 Material Design 3（MD3）的设计语言，打造现代化、干净、高效的桌面端体验。

## 阶段 1：HTML 结构与宏观布局重组
- **三栏式布局**：将现有 `.layout` 重组为标准的三栏结构：
  - **左栏 (Navigation/Sources)**：收拢原有的 Agent 列表和 Topic 列表，上下垂直排布（如上方 40% 为 Agents，下方 60% 为 Topics），对应 NotebookLM 的“来源(Sources)”面板。
  - **中栏 (Chat Stage)**：核心对话区，对应 NotebookLM 的“对话(Chat)”面板。
  - **右栏 (Studio/Settings)**：将设置面板转变为工作区（Studio），作为 Bento Grid 容器，对应 NotebookLM 右侧的功能面板。
- **HTML 调整**：解除原有 `.workspace` 的过度嵌套，将四个主要模块直接作为 `.layout` 或相应列容器的子元素，以便于使用 CSS Grid 管理 3 栏分配。

## 阶段 2：Material Design 3 视觉基础
- **色彩系统更新**：
  - **应用背景**：改为 MD3 标准的浅灰蓝色（如 `#f0f4f9`），暗黑模式使用深沉色（如 `#131314`）。
  - **面板/卡片表面**：纯白色（`#ffffff`），暗黑模式使用（`#1e1e20`）。移除生硬的黑色边框，改用极淡的描边（`rgba(0,0,0,0.05)`）和柔和的投影（Elevation Shadows）。
- **形状与圆角**：
  - 三个主面板统一使用大圆角（`24px`）。
  - 内部元素（按钮、输入框、Bento 卡片）使用 `12px` - `16px` 的圆角。
- **按钮与控件**：
  - 将 `.ghost-button`、`.accent-button` 重塑为 MD3 的 Tonal Button 和 Filled Button 样式，取消大面积边框，使用背景色块区分交互态。

## 阶段 3：右栏 "Studio" Bento Grid 深度重构
- **Bento 容器**：在右侧面板应用 `display: grid; grid-template-columns: repeat(2, 1fr); gap: 12px;`。
- **卡片化拆分**：将原线性的设置项拆解为大小不一的便当盒（Bento Box）：
  - **身份卡片 (Identity)**：横跨两列的大卡（`grid-column: span 2`），包含头像、名字、模型。
  - **参数小卡片 (Model Params)**：将 Temperature、Tokens 等参数做成 1x1 的正方形或小矩形卡片。
  - **功能性卡片（NotebookLM 灵感）**：为部分设置卡片赋予特定的浅色马卡龙背景（如浅蓝 `#e8f0fe`、浅绿 `#e6f4ea`、浅紫 `#f3e8fd`），模拟 NotebookLM 中“音频概览”、“思维导图”等工具卡片的视觉效果。
  - **系统提示词 (System Prompt)**：横跨两列的大卡片。

## 阶段 4：中栏对话区与输入框优化
- **悬浮式输入框**：改变原先吸底的 Composer 设计，将其设计为一个悬浮的药丸状/大圆角矩形，悬浮于页面底部，增加呼吸感。
- **气泡重构**：去除消息气泡的毛玻璃背景和边框，改为纯色的 MD3 气泡（Assistant 为白色带浅灰底色，User 为主题强调色或浅强调色）。
- **附件区**：附件预览以上浮缩略图的形式集成在输入框上方或内部，保持清爽。

## 阶段 5：左栏列表优化
- **列表项 (List Items)**：去除列表项默认的背景框，改为悬浮时才显示浅色背景块的 MD3 列表样式。
- **紧凑与层次**：优化 Agent 和 Topic 列表的标题（Eyebrow 字体）和操作图标，使其更像文档管理器的侧边栏。

## 阶段 6：UI 细节精修与 Bug 修复 (基于截图反馈)
- **左侧 Sidebar (导航/历史)**：
  - 修复 Topic 列表头部按钮（New Topic, Export）的挤压问题，优化纵向空间分布。
  - 修复 Topic 列表项内部的排版，确保日期、状态和操作按钮横向对齐或优雅折叠，避免现有的错位叠加。
  - 统一 Agent 和 Topic 卡片的内部滚动条样式，使其更细、更符合 MD3 规范。
- **右侧 Studio (设置/Bento)**：
  - 修复 `Profile` 卡片中输入框（Name, Model）宽度被过度压缩的问题，调整 grid 占比。
  - 美化头像上传控件（原生的 `<input type="file">` 非常破坏视觉），将其隐藏或使用自定义按钮/点击图片上传的交互。
  - 修复设置面板头部（Agent Settings 标题与 Delete 按钮）的排版冲突。
  - 调整各参数卡片（Parameters）中 input 输入框的内边距和高度，使其更协调。
  - 修复原生滚动条过粗的问题，全局应用自定义的细长圆角滚动条。
- **中间 Chat Stage (对话/输入)**：
  - 优化悬浮输入框 (Composer) 的内部比例：统一 Attach 和 Send 按钮的高度与视觉比重，修复 textarea placeholder 的垂直居中问题。
  - 优化全局留白，减弱部分卡片边框过强的突兀感。
## 阶段 7：修复头像霸屏问题 (第二次排查)
- **原因分析**：在之前的 CSS 修复中，我错误地假设了头像的类名是 `.message-avatar`。通过检查 `domBuilder.js` 发现，实际生成的头像 `<img>` 标签的类名是 `.chat-avatar`。因此之前的 CSS 规则完全没有命中目标元素。
- **修复方案**：
  - 更新 `style.css`，将控制聊天头像尺寸的 CSS 类名由 `.message-avatar` 修正为 `.chat-avatar`。
  - 确保 `.chat-avatar` 拥有 `width: 40px !important; height: 40px !important; border-radius: 50% !important; object-fit: cover !important; flex-shrink: 0;`。
  - 调整 `.details-and-bubble-wrapper` 的最大宽度，为其预留出头像和间距的空间。


## 阶段 8：引入 Material Symbols 并深度精修 MD3 视觉与排版
- **引入 Material Symbols**：
  - 在 `index.html` 中引入 Google Fonts 的 Material Symbols Outlined 字体库，这是大厂级 MD3 设计的基础。
- **左侧 Sidebar 深度修复 (基于截图 3)**：
  - **灾难级的 Topic 悬浮菜单**：目前 Hover 时出现的 `Rename`, `Mark Unread` 等按钮变成了巨大的白色方块堆叠，完全遮挡了列表项。方案：将这些操作全部替换为极简的 Material 图标（如 `edit`, `mark_email_unread`, `delete` 等），并使用绝对定位加半透明背景的 `display: flex; flex-direction: row;` 横向排布。
  - **标题按钮**：把 `New Topic` 和 `Export` 替换为带图标的IconButton或精简文字（如 ➕ New）。
- **核心组件的图标化 (Iconography)**：
  - **Composer (输入框)**：将左侧的 `Attach` 文字替换为经典的曲别针图标 `attach_file`，将右侧的 `Send` 文字替换为纸飞机图标 `send`。
  - **全局 Header**：将右上角的 `Settings` 和 `Theme` 文字替换为 `settings` 齿轮和 `dark_mode`/`light_mode` 图标，保持顶部栏极致干净。
  - **Bento Card Headers**：在各个卡片标题（如 Profile, System Prompt）旁边增加细微的辅助图标，增强模块认知。
- **留白与视觉平衡 (Spacing & Typography)**：
  - **空状态 (Empty State)**：目前的 `No messages yet...` 太生硬。需要把它居中，加上一个柔和的聊天气泡插画图标，并使用更淡的次级文本颜色 (`var(--muted)`)。
  - **输入框调优**：稍微压低一点悬浮输入框的高度，让整个药丸形状更修长优雅。


## 阶段 9：跃升至 Material 3 Expressive (去线留白、非对称与纯粹色块)
- **解决图标加载失败问题 (Fallback 兜底)**：
  - 截图显示 `send` 按钮渲染为了文字，这通常是因为网络原因导致 Google Fonts 的 Material Symbols 没有加载成功。方案：为了绝对稳定，我们将几个核心图标（`Send`, `Attach`, `Settings`, `Theme`, `Add`, `Tune`, 空状态图标等）直接替换为 **内联 SVG (Inline SVG)**。
- **无边框化 (Borderless) 与去线留白**：
  - 贯彻“去线留白”规范：移除全局所有的 `border: 1px solid var(--line)`，包括所有面板 (Panels)、卡片 (Cards)、输入框 (Inputs) 和主题头部 (Header)。
  - 深度依赖 **动态对比 (Dynamic Contrast)**：通过底层 `--paper` 颜色与表层 `--panel` 颜色的极微弱差异（如白与浅灰），辅以弥散的 `--shadow-elevation-1`（流光阴影），来确立视觉层级。
- **超大圆角与自适应布局 (Shape Scale & Adaptive)**：
  - **Panels 和 Composer**：圆角从现有的 `24px` 激进提升到 **`32px`** (Fully Circular 风格)，让界面呈现“流体”的亲和感。
  - **内部 Cards 和 Lists**：圆角提升到 **`20px`**，使所有可交互元素都像一个浑圆的果冻块。
  - **左侧导航栏重组 (Navigation Suite)**：不再使用两个僵硬的白色 Panel 包裹 Agents 和 Topics。我们将去除这两个外层容器的白色背景和阴影，让列表项直接悬浮在底色 `--paper` 上（或者给每个列表项独立的胶囊背景），这才是真正的“以内容为容器”。
- **右侧 Bento Grid 的极致表现力 (Expressive)**：
  - 加深马卡龙背景色的饱和度/亮度对比，去除所有多余的 padding 留白（让色块更纯粹）。
  - 让每个便当盒成为一个独立漂浮的、完全没有边线的“岛屿”。
- **输入框交互 (Predictive/Soft Interaction)**：
  - 所有的 Input 失去默认的描边，在平常状态下仅仅是一块微弱的凹陷色块 (`rgba(0,0,0,0.03)`)，只有 `focus` 时才会泛起主题色光晕 (`box-shadow`)。

1. **无需更改主进程逻辑**：仅需修改 `index.html` 和 `style.css`，保持现有的 DOM ID 不变，确保 `renderer.js` 能够正常绑定事件。
2. **渐进式实施**：先修改外层 Layout 和基础 CSS 变量，再深入重构右侧 Bento Grid，最后精修对话和输入框样式。
3. **兼容性**：保留深色/浅色模式切换的 CSS 变量机制。

此计划旨在不破坏现有核心功能与 IPC 交互的前提下，通过纯前端视图层的重构，大幅提升 VCPChatLite 的现代感与专业度。
